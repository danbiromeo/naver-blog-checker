# PRD: 네이버 블로그 검사기

## 1. 목적

특정 네이버 블로그의 전체 글을 크롤링하여 포스팅 빈도를 분석하고, 네이버 검색 노출 여부를 파악한다.
빈도가 높은 블로그가 미노출률이 높은지 검증하는 것이 주요 목적.

---

## 2. 입력

- 네이버 블로그 주소 **여러 개** 입력 가능 (한 줄에 하나씩)
```
https://blog.naver.com/blogid1
https://blog.naver.com/blogid2
https://blog.naver.com/blogid3
```
- 글 주소를 넣어도 자동으로 블로그 주소로 변환
  - 예: `blog.naver.com/blogid/224245623058` → `blog.naver.com/blogid`
- 검사 기간 선택: **최근 7일 / 최근 15일(기본값) / 최근 1달 / 최근 3달 / 전체**

---

## 3. 기능 (2단계 분리)

### 3-1. 📊 빈도 검사 (STEP 1 — 빠름)

크롤링만 수행. 노출 검사 없이 빈도 분석 결과를 바로 확인.

- 블로그의 모든 글 수집 (제목, 작성일, URL)
- 기간 필터링 적용 (선택한 기간에 해당하는 글만 분석)
- 블로그별 메타 정보 저장: 전체 글 수, 첫 포스팅일

**빈도 지표:**
- 일당/주당/월당 평균 빈도
- 최근 30일 포스팅 수
- 월별 포스팅 수 (막대 차트)

### 3-2. 🔍 노출 검사 (STEP 2 — 느림, 선택적)

빈도 검사 완료 후, 블로그별 탭에서 **개별 실행**. 원하는 블로그만 검사 가능.

**검색 URL 형식:**
```
https://search.naver.com/search.naver?ssc=tab.nx.all&where=nexearch&query={URL인코딩된 "제목"}&sm=tab_dgs&qdt=1
```

**판정 기준:**
- 검색 결과 HTML에 해당 블로그 ID(예: `hhncpfajcy750`)가 포함 → **노출**
- 포함되지 않음 → **미노출**

**속도 제한:**
- 요청 간 1.5초 딜레이 (네이버 차단 방지)
- 진행률 프로그레스 바 + 현재 검사 중인 글 제목 표시
- 중지 버튼으로 검사 중단 가능 (검사 완료된 것까지만 저장)

### 3-3. 빈도-노출률 상관관계 분석 (노출 검사 완료 시)

- 주간별 포스팅 수 / 노출 / 미노출 / 노출률 테이블
- 산점도 차트: X축 = 주간 포스팅 빈도, Y축 = 노출률
- 빈도 구간별 평균 노출률 요약 (예: 주 1~2회 → 85%, 주 3~4회 → 52%)

## 5. 결과 내보내기

### 비교 요약 CSV
| 열 | 내용 |
|----|------|
| 블로그ID | 블로그 아이디 |
| 전체 글 수 | 블로그 전체 포스팅 수 |
| 첫 포스팅일 | 가장 오래된 글의 작성일 |
| 기간 글 수 | 선택 기간 내 글 수 |
| 일당 빈도 | 일평균 포스팅 수 |
| 주당 빈도 | 주평균 포스팅 수 |
| 노출 | 노출 글 수 (검사 시) |
| 미노출 | 미노출 글 수 (검사 시) |
| 노출률(%) | 노출률 (검사 시) |

### 전체 포스팅 CSV
| 열 | 내용 |
|----|------|
| 블로그ID | 블로그 아이디 |
| 제목 | 글 제목 |
| 작성일 | YYYY-MM-DD |
| 노출여부 | 노출 / 미노출 / - (미검사) |
| URL | 글 링크 |
| 검색확인 | 네이버 검색 확인 링크 (검사 시) |

## 6. 기술 스택

| 항목 | 선택 |
|------|------|
| 언어 | Python |

---

## 7. 제약사항

| 항목 | 내용 | 대응 |
|------|------|------|
| 네이버 차단 | 빠른 연속 요청 시 IP 차단 가능 | 요청 간 1.5초 딜레이 |
| 노출 검사 시간 | 글 100개 기준 약 3~5분 | 빈도/노출 검사 분리, 중지 버튼 |
| 네이버 HTML 변경 | 검색 결과 구조 변경 가능 | 블로그 ID 문자열 매칭으로 단순화 |
| 새로고침 시 초기화 | session_state 날아감 | CSV 다운로드로 결과 보존 |
| 날짜 형식 혼합 | "20시간 전", "2026. 4. 9." 등 | parse_naver_date로 통일 변환 |
